## 第一章　程式功能清單

### 1.1　清單說明

本章彙整台鐵 AI 客服系統全部程式之功能對照清單，作為程式設計書之總索引。清單中每一支程式均 1:1 對應系統分析報告書（#02）所定義之功能需求（FR），並歸屬於程式設計規格報告書（#03）所規劃之軟體模組（M）。三份文件之編號體系一致，確保需求可追溯至程式碼層級：

- **程式代號（P-{模組}-NN）**：本書之程式設計單元，每支程式含類別圖、核心函式、輸入輸出規格與關鍵演算法，詳見第二章以後各節。
- **功能需求編號（FR-{模組}-NNN）**：對應 #02 系統分析報告書第二章至第八章所定義之功能需求規格。
- **軟體模組編號（M-{模組}-NN）**：對應 #03 程式設計規格報告書第五章所定義之模組設計規格。
- **測試案例編號（TC-{模組}-NNN）**：與 FR 編號 1:1 對應，詳見測試計畫書（#05）。

全系統共計 **45 支程式**，涵蓋 10 大子系統，對應 48 項功能需求與 37 個軟體模組。程式代號中，部分模組因功能需求合併實作（如 FR-LLM-002 與 FR-LLM-003 共用 P-LLM-02），故程式數量略少於功能需求總數。所有程式均以 Python 3.11+（後端）或 React 18+ / Next.js（前端）開發，技術堆疊版本鎖定詳見 #06 軟硬體清單及系統架構圖 §6.4。

### 1.2　程式功能清單

#### 一、智慧語音應答系統（AI-IVR）— 6 支程式

本子系統部署於 AI-IVR 伺服器（16 核心 ≥2.2GHz ×2，DDR5 64GB+），以 SIP/RTP fork 方式被動接收語音流（AD-007），語音處理鏈路全程於地端完成。

| 程式代號 | 程式名稱 | 所屬功能需求 | 所屬模組 | 簡述 |
|---------|---------|-------------|---------|------|
| P-IVR-01 | SIP Gateway 與 CTI 整合程式 | FR-IVR-001 語音通道與 CTI 整合<br>FR-IVR-002 智慧來電應答與引導 | M-IVR-01 | PJSIP 通話管理、RTP 收發、≥30 埠通道控制、STT→LLM→TTS Pipeline 串接 |
| P-IVR-02 | 多輪對話引擎程式 | FR-IVR-003 多輪對話與引導機制 | M-IVR-02 | Redis 對話上下文管理、Prompt 模板設計、對話狀態機控制 |
| P-IVR-03 | 轉接管理程式 | FR-IVR-004 串接真人客服 | M-IVR-03 | 對話摘要生成、SIP REFER 轉接、知識輔助推送 |
| P-IVR-04 | DTMF 偵測與動態選單程式 | FR-IVR-005 按碼偵測與動態選單 | M-IVR-04 | RFC 2833 + SIP INFO 雙模 DTMF、TTS 選單合成 |
| P-IVR-05 | 滿意度調查程式 | FR-IVR-006 滿意度調查 | M-IVR-05 | 調查流程控制、結果寫入 PostgreSQL |
| P-IVR-06 | 通話防護程式 | FR-IVR-007 防護機制 | M-IVR-06 | 敏感詞正則引擎、API Rate Limiter、對話逾時 Timer |

STT 採 CPU 推論（AD-003），30 路 faster-whisper 串流辨識約使用 9 個 CPU 核心，部署於 AI-IVR 伺服器。

| P-STT-02 | 意圖辨識程式 | FR-STT-002 語音內容分類與意圖辨識 | M-STT-02 | 意圖分類模型、路由分派邏輯 |

#### 三、語音合成系統（TTS）— 2 支程式

TTS 同樣採 CPU 推論（AD-003），每路約 0.2 CPU 核心，首包延遲 ≤300ms。Coqui XTTS-v2 為主要引擎，備案 MeloTTS / Piper 已預留介面切換機制。

| 程式代號 | 程式名稱 | 所屬功能需求 | 所屬模組 | 簡述 |
|---------|---------|-------------|---------|------|

| P-TTS-02 | 動態選單合成程式 | FR-TTS-002 動態選單語音生成 | M-TTS-02 | 選單模板引擎、SSML 標記處理 |

| P-LLM-02 | 語意理解與對話引擎程式 | FR-LLM-002 語意理解與問答生成<br>FR-LLM-003 上下文關聯與連續對話 | M-LLM-02 | RAG Pipeline（檢索→Rerank→Prompt 組裝→生成→Citation 抽取）、信心度計算與三層防線邏輯（詳見 #02 §2.12）、多輪對話上下文 Prompt 管理 |
| P-LLM-03 | 文字處理引擎程式 | FR-LLM-004 關鍵詞識別與文字修正 | M-LLM-03 | 同義詞擴展字典、敏感詞過濾規則引擎、錯別字修正（Levenshtein + 字典） |

| P-LLM-05 | 後送案件分析程式 | FR-LLM-008 後送案件分析 | M-LLM-05 | 自動分類 Prompt 設計、優先級判定規則、情感分析 Pipeline、正確率 ≥85% |

> P-LLM-02 合併實作 FR-LLM-002 與 FR-LLM-003，因語意理解與上下文關聯在推論階段共用同一 Prompt 組裝與 KV Cache 管理流程，拆分將導致重複建立對話上下文。
