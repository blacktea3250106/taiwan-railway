### 2.9 語音整合架構與 SIP/RTP 信令流程
- E1 → VoIP Gateway → MES → RTP fork → AI-IVR 完整信令流程圖（信令層級）

- E1 通道容量分析：30 B-channel 中 AI-IVR 接收 fork 副本，不佔用 B-channel
- 行政交換機路徑：類比線路 → VoIP Gateway（G.711）→ MES → AI-IVR
- WebCall SBC 雙備援 failover：SIP UA 支援雙 SBC 註冊，自動偵測切換 ≤5 秒
- 預估頁數：8 頁（含信令流程圖拉頁）
- 防禦 ATK：ATK-005（E1/RTP fork）、ATK-006（行政交換機）、ATK-007（WebCall SBC failover）

- TC-ADM-001 ~ TC-ADM-005：系統參數、帳號權限(AD)、監控儀表板、日誌稽核、備份回復（FR-ADM-001~005）
- 預估頁數：12 頁
- 防禦 ATK：ATK-023（後送系統整合邊界測試）

### 5.8 壓力測試計畫
- 情境一：語音通道壓力 — ≥30 路同時 SIP 通話，持續 30 分鐘，監控 CPU/RAM/延遲（TC-IVR-001）
- 情境二：數位通道壓力 — ≥30 路同時 WebSocket 連線，持續 30 分鐘（TC-AGT-001）
- 情境三：尖峰模擬 — 日量 2,334 通（尖峰 2 倍），驗證系統不降級
- 情境四：LLM 併發 — 30 路同時推論，回應 ≤5 秒（TC-LLM-001）
- 情境五：HA Failover — 主機故障切換，驗證 ≤5 秒恢復、Session 不遺失
- 測試工具：SIPp（SIP 壓力）、k6/Locust（HTTP 壓力）、自製腳本（LLM 併發）
- 通過標準：所有情境下 KPI 不低於契約要求值
- 預估頁數：8 頁
- 防禦 ATK：ATK-009（春節 2~3 倍話務量驗證）、ATK-014（HA 切換驗證）

### 5.9 AI 模型驗證計畫
- IVR 應答正確率：200 題（台鐵提供），正確率 ≥85%，計算公式 (完全正確×1+部分正確×0.5)÷總次數×100%
- Agent 應答正確率：200 題（台鐵提供），正確率 ≥85%，同上公式
- STT 辨識率：200 音檔（中英 4:1、≤150s），辨識率 ≥85%，計算公式 100%-(錯誤或缺漏字數/總字數)×100%
- 後送分類正確率：200 筆真實後送案件，正確率 ≥85%，計算公式 (AI 正確分類數/總案件數)×100%
- 量測起訖點定義：IVR = 旅客說完（VAD 偵測語音結束）至系統回覆首字播出；Agent = 旅客送出訊息至首段文字呈現
- 「部分正確」評分判定標準：建議由台鐵與廠商共同評分（待釐清#7）
- 預估頁數：8 頁
- 防禦 ATK：ATK-008（正確率判定/部分正確評分機制）

### 5.10 弱點掃描計畫
- 掃描工具：Nessus / OpenVAS（或同等級工具）
- 掃描範圍：全部對外服務端口、Web 應用程式、API 端點
- 掃描頻率：整合測試完成後全面掃描 1 次、上線前複掃 1 次
- 弱點分級：CVSS 3.1，Critical/High 修復後方可上線
- 預估頁數：4 頁
- 防禦 ATK：ATK-002（資安防護基準查核）

### 5.11 滲透測試計畫
- 方法論：OWASP Testing Guide v4 + OWASP Top 10（2021）
- 測試範圍：Web 前端、API 端點、SIP 信令、WebSocket、SMS 介面
- 測試項目：注入攻擊、認證繞過、越權存取、XSS/CSRF、Prompt Injection
- 測試團隊：具 CEH/OSCP 認證之資安人員
- 報告格式：弱點描述、風險等級、重現步驟、修復建議
- 預估頁數：5 頁
- 防禦 ATK：ATK-002（CNS 27001 技術驗證）、ATK-004（OWASP Mobile Top 10）

### 5.12 源碼檢測計畫
- 檢測工具：SonarQube / Fortify / Semgrep（或同等級）
- 檢測範圍：自行開發之全部原始碼，排除第三方元件原始碼
- 弱點分級：Critical/High 修復後方可上線
- 開源元件漏洞掃描：OWASP Dependency-Check / Snyk
- 預估頁數：4 頁
- 防禦 ATK：ATK-002（資安檢測完整性）

### 5.13 個資保護測試計畫
- 去識別化效果驗證：測試 STT 輸出是否正確遮罩身分證/電話
- 存取權限隔離驗證：不同角色存取個資欄位之權限檢查
- 個資存取日誌驗證：所有存取行為是否完整記錄
- 個資加密儲存驗證：資料庫欄位級加密之有效性
- 預估頁數：4 頁
- 防禦 ATK：ATK-001（個資去識別化效果驗證）、ATK-003（錄音使用合規驗證）

### 5.14 測試排程與台鐵配合事項
- 單元測試：D+100~150（開發團隊自行執行）
- 整合測試：D+150~200（含既有系統介接測試）
- 壓力測試 + 資安檢測：D+200~220
- AI 模型驗證：D+210~230（台鐵提供題庫/音檔）
- UAT：D+220~240（台鐵參與）
- 台鐵配合：提供測試題庫(200 題×2 + 200 音檔 + 200 筆後送案件)、測試環境網路配合、UAT 參與人員
- 預估頁數：4 頁
- 防禦 ATK：ATK-013（時程能否跑完）

**#05 合計預估頁數：106~112 頁**
