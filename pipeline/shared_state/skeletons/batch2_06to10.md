## #08 系統上線執行計畫書

### 8.1 上線策略與總體規劃
- 上線策略：灰度上線（10% → 30% → 100%），而非一次全量切換
- 上線時程：預計 D+200 開始灰度、D+220 全量上線、D+240 完成驗收測試
- 回滾判定標準：可用性 <95% 或正確率 <70% 或 MTTR >4hr 時觸發回滾
- 回滾方案：AI-IVR 停止接聽 → 全部流量回歸人工客服，不中斷服務（合規紅線#7）
- 預估頁數：4 頁
- 防禦 ATK：ATK-008（AI 回答錯誤風險的分階段控制）、ATK-009（尖峰時段避免上線）

### 8.2 上線前置作業檢核表
- 硬體就緒：設備上架、電力/網路/散熱確認（引用 #06 §6.2~6.5）
- 軟體部署：Docker Compose 全服務啟動驗證（引用 #03 §3.9）
- 知識庫匯入：種子資料匯入完成、embedding 建置完成、覆蓋率達標（引用 #02 §2.18）
- 介接驗證：10 套既有系統介接逐項測試通過（引用 #02 §2.14）
- 帳號建置：AD 整合測試、RBAC 角色設定、全員帳號開通（FR-ADM-002）
- 備份機制：首次全備完成、排程啟動確認（FR-ADM-005）
- 監控告警：Prometheus/Grafana 指標正常、告警通知鏈測試（FR-ADM-003）
- 教育訓練：值機員 ≥36hr、管理者 ≥18hr 完成（引用 #04）
- 預估頁數：6 頁
- 防禦 ATK：ATK-019（API 介接就緒確認）、ATK-020（知識庫就緒確認）

### 8.3 網路與機房配置執行計畫
- 機房進場時程：設備交貨 → 上架 → 佈線 → 電力測試 → 開機
- VLAN 設定：管理 / 服務 / 資料三段隔離（引用 #06 §6.7）
- 防火牆規則：各 VLAN 間存取控制清單（ACL）
- E1 / SIP Trunk 線路測試（引用 #06 §6.8）
- HA Keepalived + VIP 配置與切換測試（引用 #02 §2.13）
- 預估頁數：5 頁
- 防禦 ATK：ATK-005（E1 線路就緒）、ATK-012（機房電力散熱）、ATK-014（HA 網路配置）

### 8.4 灰度上線階段規劃
- 階段一（D+200~D+207）10% 流量：僅限 WebCall 通道，AI-IVR 處理簡單 FAQ 查詢
  - 監控重點：正確率、回應時間、轉人工比例
  - 每日檢討會議，逐日決定是否擴大
- 階段二（D+208~D+214）30% 流量：加入 PSTN E1 部分通道 + AI-Agent 文字客服
  - 監控重點：併發穩定性、GPU 使用率、Redis Session 正確性
- 階段三（D+215~D+220）100% 全量：全通道啟用，含行政分機、SMS
  - 監控重點：全面 KPI 達標驗證
- 每階段 Go/No-Go 判定標準與責任人
- 預估頁數：6 頁
- 防禦 ATK：ATK-009（尖峰期間不排灰度上線）、ATK-010（轉人工比例監控）

### 8.5 資料遷移計畫
- 知識庫種子資料匯入：FAQ / SOP / 規章 / 時刻表 / 票價表（待釐清#3 知識庫範圍）
- 歷史對話資料：評估是否遷入（視台鐵需求）
- 帳號資料：AD 同步驗證
- 資料遷移驗證：筆數比對、抽樣正確性驗證
- 預估頁數：3 頁
- 防禦 ATK：ATK-020（知識庫資料完整性）

### 8.6 上線風險管理
- 風險一：API 介接異常 → Mock 降級，AI 回覆「此功能暫時維護中，請洽專人」
- 風險二：GPU 過載 → 自動降級至單實例模式，拉長回應但不中斷
- 風險三：知識庫覆蓋不足 → 調低信心度閾值，增加轉人工比例（引用 #02 §2.12 三層防線）
- 風險四：上線初期客訴量增 → 督導即時監控 + 密語指導（FR-WS-004）
- 緊急聯絡人清單：台鐵窗口、廠商 7×24 待命人員、電務處值班
- 預估頁數：4 頁
- 防禦 ATK：ATK-008（AI 錯誤回答的上線期應對）、ATK-011（Coqui TTS 上線驗證）

### 8.7 上線驗收標準
- KPI 驗收指標對照（引用 #02 §2.10 非功能需求）：
  - IVR 正確率 ≥85%（200 題）、回應 ≤5,000ms、≥30 埠
  - Agent 正確率 ≥85%（200 題）、回應 ≤5,000ms、≥30 埠
  - STT 辨識率 ≥85%（200 音檔，中英 4:1）
  - QI 15min 錄音 → ≤5min 完成
  - 分類正確率 ≥85%（200 筆）
  - 可用性 ≥99%/季
- 驗收測試題庫由台鐵提供（待釐清#7 正確率判定標準）
